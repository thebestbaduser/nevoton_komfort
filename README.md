# Nevoton Komfort Sauna

Кастомная интеграция Home Assistant для управления контроллером сауны Nevoton Komfort / KOMFORT-WF по локальному HTTP API.

Интеграция поддерживает:
- климат-контроль сауны
- освещение и диммер
- переключатели питания, вентилятора, парогенератора и отложенного старта
- сенсоры температуры, влажности, статуса и таймеров
- числовые параметры таймеров и целевой влажности

## Совместимость

- Home Assistant: `2024.1.0+`
- API устройства: `NEVOTON REST API v1.0`
- Проверенный контроллер: `KOMFORT-WF`, firmware `3.5`

## Что уже учтено в интеграции

Интеграция уже умеет работать с особенностями контроллера:
- нестандартные HTTP-заголовки устройства
- вложенный `specific`-ответ (`switchers`, `timers`, `sensors`, `dimmers`)
- адреса вида `IP`, `IP:port` и `http://IP:port`
- кратковременные таймауты и задержки ответа после команд `set`

Если контроллер применяет изменения не мгновенно, Home Assistant может показывать новое состояние с небольшой задержкой. Интеграция старается сгладить такие провалы и не "откидывать" сущности в `Unavailable` при кратковременных сбоях.

## Установка через HACS

1. Откройте HACS в Home Assistant.
2. Нажмите на меню в правом верхнем углу.
3. Выберите `Custom repositories`.
4. Добавьте репозиторий:

   ```text
   https://github.com/thebestbaduser/nevoton_komfort
   ```

5. Выберите категорию `Integration`.
6. Установите `Nevoton Komfort Sauna`.
7. Полностью перезапустите Home Assistant.

## Настройка

1. Откройте `Настройки -> Устройства и службы`.
2. Нажмите `Добавить интеграцию`.
3. Выберите `Nevoton Komfort Sauna`.
4. Укажите:
- адрес контроллера
- пароль устройства

Поле адреса поддерживает такие форматы:
- `192.168.1.50`
- `192.168.1.50:8877`
- `http://192.168.1.50:8877`

Если у вас Home Assistant работает в той же локальной сети, лучше указывать именно локальный адрес контроллера, а не внешний проброшенный порт.

## Сущности

### Climate

- `climate.nevoton_komfort_sauna`

Возможности:
- включение и выключение нагрева
- установка целевой температуры
- отображение текущей температуры

### Light

- `light.nevoton_komfort_light`

Возможности:
- включение и выключение света
- управление яркостью через `Light_dimmer`

### Switch

- `switch.nevoton_komfort_main_power`
- `switch.nevoton_komfort_fan`
- `switch.nevoton_komfort_humidity`
- `switch.nevoton_komfort_timer_offset`

### Sensor

- текущая температура
- текущая влажность
- статус
- оставшееся время нагрева
- время отложенного старта

### Number

- целевая влажность
- таймер нагрева
- таймер отложенного старта

## Особенности отклика

Контроллер не всегда подтверждает изменение состояния мгновенно. На практике это проявляется так:
- ручное изменение на самом контроллере может долетать в HA с задержкой в несколько секунд
- после команды из HA контроллер может еще некоторое время отдавать старое состояние
- во время обновления таймеров и режимов контроллер иногда отвечает медленнее обычного

Сейчас интеграция:
- опрашивает контроллер чаще, чем в ранних версиях
- удерживает последнее подтвержденное состояние после успешной команды
- повторяет запись при единичном таймауте
- переживает кратковременные неудачные опросы без мгновенного перехода в `Unavailable`

## Безопасность

Не рекомендуется публиковать контроллер напрямую в интернет.

Причины:
- API работает по обычному `HTTP`, без `HTTPS`
- хэш пароля передается в query string
- устройство рассчитано в первую очередь на локальную сеть

Безопаснее так:
- держать контроллер только в локальной сети
- давать удаленный доступ к Home Assistant через VPN
- сменить пароль по умолчанию `admin`
- не оставлять постоянно проброшенный наружу порт устройства без необходимости

## Устранение неполадок

### Устройство не добавляется

- Проверьте адрес и порт.
- Убедитесь, что контроллер доступен из сети, где работает Home Assistant.
- Если контроллер слушает нестандартный порт, укажите его явно: `IP:PORT`.

### Сущности обновляются с задержкой

- Это может быть особенностью самого контроллера.
- Дайте устройству несколько секунд после изменения состояния.
- Убедитесь, что используете актуальную версию интеграции.

### Сущности становятся `Unavailable`

- Проверьте качество сети между HA и контроллером.
- Если используется внешний проброс порта, проверьте NAT/маршрутизацию и задержки.
- По возможности переведите интеграцию на локальный IP контроллера.

### Неправильные значения или пустые сенсоры

- Проверьте, что устройство реально поддерживает `specific`-канал.
- Сравните ответ `/help/specific` с тем, что ожидает интеграция.
- Если прошивка отличается, приложите пример ответа `/get/m2m/inputs?type=specific&number=All&hash=...` в issue.
