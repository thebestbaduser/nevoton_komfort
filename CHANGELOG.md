# Changelog

## 1.0.10

- Повышена устойчивость к медленным ответам контроллера.
- Увеличен таймаут чтения.
- Интеграция дольше сохраняет последнее корректное состояние при серии кратковременных неудачных опросов.

## 1.0.9

- Добавлено удержание последнего подтвержденного состояния после команды `set`.
- Убран эффект "отпрыгивания" переключателей назад, пока контроллер еще не начал отдавать новое значение.

## 1.0.8

- Улучшен отклик после команд из Home Assistant.
- Убран лишний блокирующий refresh из цепочки пользовательской команды.
- Добавлен повтор записи при единичном `Connection timeout`.
- Сокращен интервал опроса для более быстрого получения ручных изменений с контроллера.

## 1.0.7

- После успешной команды локальное состояние обновляется сразу.
- Кратковременные ошибки чтения после `set` больше не переводят сущности в `Unavailable` мгновенно.

## 1.0.6

- Исправлен разбор вложенного `specific`-ответа контроллера.
- Добавлена поддержка адресов вида `host:port` и `http://host:port`.
- Исправлено формирование ссылки на веб-интерфейс устройства.

## 1.0.5

- Для чтения `specific` добавлен fallback с `number=All` на `number=0`.
- Статус больше не подменяется значением `0`, если данных фактически нет.

## 1.0.4

- Добавлено автоматическое сопоставление ожидаемых имен параметров с реальными ключами контроллера.
- Чтение и запись `specific`-параметров переведены на общий слой разрешения имен.

## 1.0.3

- Обновлена версия пакета после исправления обработки API-запросов.
- Подготовлена корректная базовая точка для последующих фиксов совместимости.
